Local GitHub Workflow With Command-Line Git & Docker

Overview

The purpose of this tutorial will be to explore how to incorporate command-line git into your local workflow.

  • I'll start this exploration with a brief overview of the software development tools I use in conjunction with GitHub (that make my workflow possible).

  • Next, I'll briefly explain how to fork a repo, and elaborate on some the benifits of working locally from your own fork (later, I will also explore how to use the command line to keep your fork in sync and contribute back to the upstream repo).

  • I'll continue by exploring how to set up your local repo from your fork and how to connect your local repo to your live GitHub repo all from the command line.

  • After this initial setup, I will explore the command line workflow for managing a project locally with git.

  • After implementing the code for a feature, I will examine how to test your code locally (both in broswer and by using Docker to run super-linter from the command lin) before pushing the code to GitHub.

  • And last but not least, I will explore how to use the command line to merge my local branch back into main.
