Environment and Workflow

The point of this project is to get hands on experience using Git/GitHub by collaborating with a group to create and launch a website. Each team member is responsible for contributing to the project using any Git/Github environment and workflow they are most comfortable with.

Production Website

We will use a fork of 'websitedemo' as the seed code for our project. We will develop a site together by reuseing, editing, modifying, deleting, or doing anything we want to with the forked seed code. For our groups website we chose to modify 'websitedemo' by each creating a GitHub tutorial Page.

Project Roadmap

The first three weeks of the term prepared our team to launch our group project by learning foundational GitHub skills. We launched the project on week four and will work on it for the remainder of the term. Our project will incorporate new material/skills each week. The final week we present our project.

Repository Requirements

Our production repo must include: an active license, a README.md file, a CONTRIBUTING.md file, a project page (this site), a bug reports project board, a Super-linter workflow, and all project assets.
